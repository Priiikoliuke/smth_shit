# Пересказ темы видео: "Память, адресация и защита памяти в современных компьютерах"

Видео посвящено **как компьютеры хранят, читают и защищают** данные в памяти, с технической и визуальной стороны.

# Память и CPU

- После CPU память — следующий по важности компонент.
- Память состоит из сетки маленьких цепей (транзисторы/конденсаторы), каждая из которых хранит 1 или 0.
- Каждая цепь имеет уникальный **адрес**, что позволяет CPU считывать или записывать данные.

# Адреса и слова

- Один бит обычно состоит из 8 цепей (1 байт).
- С помощью байта можно представлять до 256 значений.
- CPU работает с памятью через инструкции:
  - `Load` — чтение из памяти
  - `Store` — запись в память

# Процессы и конкуренция

- Одновременная работа нескольких программ вызывает конкуренцию за CPU и память.
- Программы должны быть загружены в память перед выполнением, так как CPU не работает напрямую с диском.
- **Пространство адреса процесса** — область памяти, выделенная процессу.
- Процесс не должен иметь доступ к памяти другого процесса.

# Защита памяти

- Без защиты памяти процессы могли бы:
  - Читать данные других процессов
  - Изменять данные или код других процессов
  - Изменять код ОС
- Проверка доступа к памяти в ПО слишком медленная и ресурсоёмкая.
- Аппаратная проверка через регистры:
  - **Базовый регистр (Base)** — минимальный адрес памяти процесса
  - **Закрытый регистр (Limit)** — размер адресного пространства
- Логика проверки:
  - Адрес должен быть ≥ Base
  - Адрес должен быть < (Base + Limit)
- Используются бинарные сравнения и логические ворота (AND), чтобы проверить адрес.

# Архитектура и визуализация

- CPU видит только поток адресов через адресную шину.
- Основная память не знает, как генерируются адреса.
- Использование двух регистров (Base + Limit) аппаратно ограничивает доступ процесса к памяти.
- Выход логических операций определяет, корректен ли адрес.

# Резюме безопасности

- Адресное пространство процесса — святое.
- Процесс не может читать/писать в память другого процесса.
- Защита памяти реализуется аппаратно для высокой производительности.
